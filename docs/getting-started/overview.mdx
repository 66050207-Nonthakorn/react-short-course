# Overview

> แนะนำว่าให้อ่านซักนิดนึง จะได้ไม่งงตอนทำเรียนบทต่อๆไป ^^

ในหน้านี้จะอธิบายเบื้องต้นถึงหลักการทำงานของเว็บว่ามีการทำงานอย่างไรและเราจะมี React ไปเพื่ออะไร

## How traditional website works

ก่อนจะเข้าใจว่าเว็บในสมัยนี้ทำไมถึงเป็นอย่างนั้นหรือแบบนี้ ต้องเข้าใจการทำงานในอดีตกันก่อน

โดยเมื่อก่อนนั้นการที่เราเข้าซักเว็บไซต์นึงเช่น [Google](https://google.com) คอมพิวเตอร์เราจะส่งข้อมูลขอหน้าเว็บไซต์จาก Server ของเว็บนั้นๆ
(คิดซะว่าเป็น Airdrop ของ iPhone ในสมัยนี้ละกัน)

:::note
Server ก็แค่คอมพิวเตอร์เครื่องหนึ่ง แต่ได้มีการตั้งค่าหรือติดตั้งบางอย่างทำให้ Server ไม่เหมือนกับคอมพิวเตอร์ทั่วๆไป
:::

หลังจากขอข้อมูลไป Server จะส่งไฟล์นึงมาให้เรา เป็นรูปแบบไฟล์ที่มีนามสกุล `.html`
และบราวเซอร์ของเรา (เช่น Google Chrome, Safari) จะมีหน้าที่นำไฟล์นั้นมาจัดรูปแบบ กลายเป็นเว็บไซต์ที่เราเห็นกัน

ในยุคต่อมาก็ได้มีการเพิ่มไฟล์ที่ส่งเข้ามาเพื่อทำให้เว็บไซต์มีสีสันและสามารถทำอะไรได้หลายๆอย่างโดยไฟล์ที่เพิ่มมานั้นเรียกว่า
Cascading Style Sheet หรือเรียกสั้นๆว่า CSS (นามสกุล `.css`) และ ไฟล์ Javascript (นามสกุล `.js`) เพื่อเพิ่ม
ความสามารถของตัวเว็บเข้าไปอีก

:::info
สรุปง่ายๆก็คือ ถ้าเราอยากเข้า Google เราก็จะต้องไปขอ Google แล้ว Google ก็จะให้ไฟล์เรามา 3 ไฟล์
บราวเซอร์ของเราก็เอาไฟล์นั้นมาจัดวางให้สวยๆ เพื่อให้เราดู
:::

โดยปกติแล้วการขอดูแต่ละหน้าก็จะเท่ากับ 1 html ไฟล์ ดังนั้นการที่เราเปิดในแต่ละหน้า
ก็จะเท่ากับการขอ Google ไปตามจำนวนหน้าที่ขอ เปิดไป 4 หน้าเว็บ ก็เท่ากับ Google ส่งไฟล์มาให้เรา 4 ครั้ง
ซึ่งถ้าเครื่อง Server ของ Google เกิดคนขอเยอะๆในเวลาเดียวกัน อาจจะทำให้ Server ไม่ตอบสนองและไม่ส่งอะไรมาเลย
หรือที่เราเรียกกันว่า เว็บล่ม
