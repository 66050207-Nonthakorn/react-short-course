# Component & Props

## Component

import componentImg from "./assets/components.png"

ใน React จะใช้หลักการ Component-based architecture หรือพูดง่ายๆว่าเราจะมองทุกอย่างในหน้าเว็บที่เรา Design
แบ่งออกเป็นส่วนต่างๆ แล้วนำมาประกอบกัน

<img src={componentImg} />

ตัวอย่าง Syntax ของ React ในการเขียน 1 Component React จะใช้ Function เป็นตัวแทน

และ Function นั้นๆ จะ return ค่าเป็น element ใน HTML

```tsx
function MyText() {
  return <h1>Hello World !</h1>
}

// Lambda
const MyText = () => (
  <h1>Hello World !</h1>
);
```
---

HTML ที่ return นั้น ไม่เหมือนกับ HTML ทั่วๆไป, React เพิ่มความสามารถให้เราเขียนตัวแปร Javascript ใน HTML นั้นๆได้ตัวอย่างเช่น สร้างตัวแปรอักษรและนำไปใส่ได้

```tsx {1,4}
const name = "apple";

function MyText() {
  return <h1>Hello World {name}!</h1>
}
```

จริงๆแล้วมันมีประโยชน์มากกว่านั้นนอกจากใส่ตัวแปรเฉยๆ โดยในพูดถึงในหน้าถัดๆไป


และในตอนเรียกใช้ Component จะต่างกับการเขียนเรียกใช้ Function ทั่วๆไปโดยหน้าตาคล้ายกับ element ของ HTML
```tsx
MyText()    ❌
<MyText />  ✅
```

---

## Properties

หรือเรียกสั้นๆว่า Props เป็นการส่งข้อมูลไปให้ Component คล้ายกับการส่งค่าไปใน Function

เราจะกำหนดว่า Props จะมีค่าอะไรบ้างด้วย interface จากนั้นเรากำหนด Parameter รับ object
ชื่อ props (จริงๆแล้วชื่ออะไรก็ได้)

```tsx {1-4,6,9-10}
interface MyTextProps {
  name: string
  num: number
}

function MyText(props: MyTextProps) {
  return (
    <>
      <h1>My name is {props.name}</h1>
      <p>and i like number {props.num}</p>
    </>
  );
}
```

ซึ่งการส่งค่าเป็นแบบ attribute=value ตัวอย่างเช่น

```tsx
<MyText name="apple" num={4} />
```

โดยนอกจะส่งค่าพื้นฐานเช่นตัวเลขหรือข้อความ ยังมี Props พิเศษอีกหนึ่งอย่างโดย Props นั้นจะมี attribute ชื่อ children

โดย attribute children นั้นจะเป็น Props สำหรับรับ Component

```tsx
interface ParentProps {
  children: React.ReactNode[]
}

function Parent(props: ParentProps) {
  return (
    <div>
      {props.children}
    </div>
  );
}
```

โดยวิธีใช้งาน สมมติว่าเราอยากให้มีปุ่มอยู่ใน Component นี้

```tsx
<Parent>
  <button>Hello I am child</button>
</Parent>
```

ก็จะเหมือน HTML ทั่วๆไปที่สามารถซ้อนกันได้ แต่ Child จะอยู่ตรงไหนหรือทำอะไรเป็นหน้าที่ของเราที่ต้องกำหนด
